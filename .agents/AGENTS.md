# Agent Rules & Guidelines

- Limit the number of commits and pull requests by grouping related tasks together.
- When working on features or fixes, batch related backend and frontend changes into unified branches and PRs to keep the commit history clean.
- Use squashing to condense commits when merging.
- **NEVER merge directly into `main` automatically.** Always push the feature branch to the remote repository and wait for explicit approval from the user after they have tested the branch locally.
